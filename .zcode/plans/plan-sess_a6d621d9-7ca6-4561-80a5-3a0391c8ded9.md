## Goal
1. **Bug fix:** Color-temperature tint must only affect the viewfinder, not the whole UI.
2. **UI upgrade:** Redesign the Temperature & Exposure controls as rich rectangular panels with a multi-color hue/spectrum strip and a Kelvin (K) readout.

All changes are in **`app/src/main/java/com/example/CameraUi.kt`** only. Internal storage stays as the existing `-2..2` float (so saved-photo filter logic and capture are untouched — zero risk there); K is purely a display conversion.

---

## 1. Fix the tint bug (viewfinder-only)

Currently the tint overlay sits in the root `BoxWithConstraints` with `Modifier.fillMaxSize()` (`CameraUi.kt:304-317`), washing amber/teal over the controls, bottom deck, and buttons.

**Fix:** Move that `Box` *inside* the viewfinder container `Box` (lines 278-302), as the first child stacked above `CameraPreviewView`, and clip it with the same `RoundedCornerShape(16.dp)` so it matches the viewfinder's rounded corners. The overlay then only ever covers the live preview.

The saved-photo tint is already handled separately in `CameraViewModel.applyRetroFilter` (`CameraViewModel.kt:392`), so capture output is unaffected.

---

## 2. New `SpectrumSlider` composable (reusable for both panels)

A single custom composable used for both Temperature and Exposure, with params for gradient colors, range, ticks, and value formatting:

- **Rectangular track** drawn as a horizontal multi-color gradient bar (rounded, ~28dp tall). Custom-rendered via a `Box` with `Brush.horizontalGradient` + `onGloballyPositioned` to capture pixel width.
- **Drag + tap-to-set** via `Modifier.draggable(Horizontal)` + `detectTapGestures` (value computed from touch x → range). No dependency on Material3 slider slot APIs (avoids version risk).
- **Thumb:** a vertical white indicator notch sitting on the track, with a small **value chip** above it (e.g. `5500K` or `+1.2 EV`).
- **Tick marks** under the track at the min/center/max positions.
- **Double-tap to reset** to the default (0) — `detectTapGestures(onDoubleTap = ...)`.

## 3. Temperature panel (rectangular, colorful, Kelvin)

- Wider panel (~300dp) with header: `WHITE BALANCE` · live K readout (e.g. `3200K · Warm`) · `↺` reset hint.
- **Spectrum gradient** = realistic blackbody-ish color-temperature ramp:
  `#9F1F4B (deep magenta-red)` → `#F59E0B (amber)` → `#FDE68A (warm white)` → `#F8FAFC (neutral)` → `#7DD3FC (cyan)` → `#1D4ED8 (deep blue)`.
- **Kelvin mapping** (photographically accurate — warmer light = lower K):
  - `temp = -2` (Cool) → `9000K`
  - `temp = 0` → `5600K` (daylight)
  - `temp = +2` (Warm) → `2300K`
  - Formula `K = 5600 - (temp * 825)`, rounded to nearest 50K.
- Ticks labeled `9K` / `5.6K` / `2.3K`.

## 4. Exposure panel (matches the new style)

- Header: `EXPOSURE` · live EV readout (e.g. `+1.2 EV`) · reset hint.
- **Gradient** = brightness ramp: `#000000` → `#3F3F46` → `#A1A1AA` → `#FBBF24 (warm highlight)`.
- Range stays `-3..3 EV`. Ticks labeled `-3` / `0` / `+3`.

---

## 5. Container tweaks

- Widen the popup `Box` from `240.dp` to ~`300.dp`, slightly more vertical padding, keep the existing dark glass background (`0xE61E1E1E`) and border so it stays on-brand retro.
- Keep the existing `AnimatedVisibility` slide/fade animation and the mutual-exclusion logic (only one of temp/exp open at a time) — unchanged.
- The whole-screen tap-to-close behavior stays; the slider's own drag/tap gesture consumers will prevent drags from accidentally closing the panel.
